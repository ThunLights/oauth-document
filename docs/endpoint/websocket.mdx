# /ws

By using this websocket endpoint, OAuth can be used via websocket.

## Examples

```ts
import WebSocket from "ws";

const ws = new WebSocket("wss://oauth.thunlights.com/ws");

ws.on("open", async () => {
    ws.on("message", (message, isBinary) => {
        const data = JSON.parse(message);
        if (data.type === "heartbeat") {
            return ws.send(JSON.stringify({
                type: "heartbeat",
                content: Date.now(),
            }));
        }
    });

    ws.send(JSON.stringify({
        type: "handshake",
        applicationId: "APPLICATION_ID",
        secret: "SECRET_KEY",
    }));
});
```

Basic communication can be done with the above code.

## Message Types

Request Types
```json
{
    "type": "heartbeat",
    "content": 12345
}
```
```json
{
    "type": "handshake",
    "applicationId": "APPLICATION_ID",
    "secret": "SECRET_KEY"
}
```
```json
{
    "type": "code",
    "content": "INPUT_CODE_HERE"
}
```
```json
{
    "type": "accessToken",
    "content": "INPUT_ACCESSTOKEN_HERE"
}
```
```json
{
    "type": "refreshToken",
    "content": "INPUT_REFRESHTOKEN_HERE"
}
```

Response Types
```json
{
    "type": "error",
    "content": "ERR_CONTENT"
}
```
```json
{
    "type": "welcome",
    "content": "WELCOME_MESSAGE"
}
```
```json
{
    "type": "open",
    "content": "YOUR_APPLICATION_ID"
}
```
```json
{
    "type": "code",
    "content": {
        "accessToken": "ACCESSTOKEN",
        "refreshToken": "REFRESHTOKEN"
    }
}
```
```json
{
    "type": "accessToken",
    "content": {
        "id": 0,
        "txtId": "TXTID",
        "email": "EMAIL",
        "displayName": "DISPLAYNAME"
    }
}
```
```json
{
    "type": "refreshToken",
    "content": {
        "accessToken": "ACCESSTOKEN"
    }
}
```
